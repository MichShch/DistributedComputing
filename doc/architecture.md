# Архитектура системы распределенных вычислений

## 1. Обзор
Система состоит из трех основных компонент:
- Master (Control Service)
- Worker Agent
- CLI клиент

Master управляет агентами и задачами, Worker выполняет задачи, CLI взаимодействует с Master по API.

## 2. Логическая схема
- Master хранит состояние агентов и задач.
- Worker регистрируется в Master, отправляет heartbeat и получает задания.
- CLI отправляет команды пользователю через Master API.

Схема взаимодействия (упрощенно):
```
CLI ----REST----> Master <----REST---- Worker
             (планирование, хранение, API)
```

Сценарий выполнения задачи:
```
CLI -> Master: SubmitTask
Master -> Worker: назначение (PollTasks)
Worker -> OS: запуск процесса
Worker -> Master: TaskStatus
CLI -> Master: GetTask / GetLogs
```

## 3. Компоненты

### 3.1 Master
Ответственность:
- регистрация и мониторинг агентов
- очередь задач и планирование
- API для внешних клиентов
- хранение состояния и логов

Подсистемы:
- Agent Registry
- Task Queue
- Scheduler
- API Server (REST)
- Storage (файлы/БД)

### 3.2 Worker Agent
Ответственность:
- регистрация в Master
- получение и выполнение задач
- отправка статусов и логов

Подсистемы:
- Agent Client (API)
- Task Runner (process manager)
- Heartbeat
- Log Collector

### 3.3 CLI
Ответственность:
- управление задачами и агентами
- отображение статусов и логов

Подсистемы:
- API Client
- Output Formatter (text/json)

## 4. Хранение состояния
Минимальный набор данных:
- таблица агентов (id, status, os, resources, last_heartbeat)
- таблица задач (id, state, params, assigned_agent, timestamps)
- логи задач (stdout/stderr, метаданные)

Хранилище может быть реализовано через файл или БД.

## 5. Потоки и конкуренция
- Master обрабатывает запросы API параллельно
- Планирование задач выполняется в отдельном потоке
- Worker выполняет задачи в отдельных процессах ОС

## 6. Ошибки и восстановление
- При потере heartbeat агент помечается offline
- Задачи с агентом offline возвращаются в очередь
- Worker восстанавливает регистрацию после перезапуска
